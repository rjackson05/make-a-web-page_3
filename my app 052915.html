Basics of Web and HTML

The web is a bunch of computers that communicate with one another. When entering the web the language used is HTML. HTML stands for Hypertext Markup Language. It was created in 1990. The majority of documents on we content is HTML. Computers talk through HTTP by requesting information via a server. HTML documents contain text content that is “what you see” and markup is “how it looks”. Tags and elements are used in HTML. Elements are HTML documents are made up of HTML “elements’. Elements are either inline or block. Block elements have invisible boxes around them. Tags tell the References to other documents - e.g. images & videos

Creating a Structured Document with HTML


HTML and CSS are both languages. HTML is HyperText Markup language and CSS Cascading Style Sheets. The “tree-like structures” of HTML control the structure of the web page. CSS controls the style of how the page looks. Programmers use DOM (Document Object Model) when structuring the page. Boxes are used to help organize the pages.

Adding CSS Style to HTML

CSS is code written to control the "style" of HTML elements. This can be done by using elements and class. There are 3 methods to write CSS (Cascading Style Sheets). Method 1: Write CSS into the HTML by adding: head element and style elements. This is used for small projects Method 2: Link HTML to a separate CSS file. This adds additional steps but works well in larger projects. Method 3: Write style inline with your HTML by coding each individual HTML element. This approach can cause a lot of repeated code. You can identify a class or tag of the element. “Cascading” rules are applied buy direct matches and child elements. When there are multiple or overlapping rules then more specific rules take precedent. Inheritance is a key feature, and it relies on document trees and the hierarchy of elements on the page. An example of inheritance is h1. One common way to format CSS is to indent each property and give it its own line. This makes it easier to manage. Code, Test, Refine 1. Look for natural boxes 2. Look for repeated styles and semantic elements 3. Write your html 4. Apply styles -Go from biggest to smallest 5. Fix things It is important to verify your css code. It is also important to avoid repetition in your code. As project grow this becomes for important to keep your code easy to test, correct and build on. There are some sites to help you check your code and they are: HTML Code Validator at http://validator.w3.org/check CSS Validator at https://jigsaw.w3.org/css-validator
